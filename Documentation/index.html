Zenith Dental Template Documentation

A comprehensive guide to customize, configure, and deploy the Zenith Dental template.

Introduction

Welcome to the official developer documentation for the **Zenith Dental** website template. Zenith Dental is a premium commercial template built using standard, modular HTML5, CSS3 Custom Properties, and Vanilla ES6 JavaScript.

This template features a complete patient booking wizard, fully responsive layouts, optimized WebP images, local business SEO configurations, and WCAG AA accessibility standards, making it the perfect platform for medical and clinical practices.

Installation & Setup

Since Zenith Dental is a static HTML5/CSS3 template, setting it up is simple. No compilers or server-side installs are strictly required to preview the site.

1. Local Preview

Simply navigate to the Template/ folder and open index.html in any web browser to view the page instantly.

2. Web Server Deployment

To run the live booking scheduler and have all modules function optimally, upload the contents of the Template/ folder directly to your web server (e.g. Apache, Nginx, cPanel hosting, or static providers like Netlify, Vercel, or GitHub Pages).

Tip: If you are planning major CSS additions, editing variables directly in Template/assets/css/styles.css will update components globally without code duplication.

Design System & CSS Styling

The layout uses CSS custom variables (variables declared in the :root selector of styles.css) to maintain global styling consistency. You can change colors and fonts by updating these variables:

:root {
  /* Core Clinical Palette */
  --color-primary: #0d9488;          /* Calm clinical teal */
  --color-primary-dark: #0f766e;     /* Hover state color */
  --color-secondary: #0284c7;        /* Medical accent blue */
  --color-accent: #d97706;           /* Amber/Gold rating star color */
  
  /* Typography setup */
  --font-heading: 'Outfit', sans-serif;
  --font-body: 'Inter', sans-serif;
}

All button styles, cards, and section background colors read directly from these design system variables.

Services & Doctors Data (JS)

Instead of hardcoding doctor cards, reviews, or services in HTML, these arrays are configured inside Template/assets/js/data.js and generated dynamically. To edit them, modify the data array objects:

const dentalServices = [
  {
    id: "general",
    title: "General Dentistry",
    shortDescription: "Preventative dental care for families...",
    price: "From $90",
    duration: "45-60 mins",
    treatments: [
      "Routine Cleanings & Screenings",
      "Composite Fillings"
    ]
  }
];

The Booking calendar automatically checks which week-days each clinician is scheduled to work by checking their active schedule arrays in data.js.

Clinic Contact Information

Static business contact information (Opening hours, telephone numbers, coordinates, address text) is defined in Template/index.html. We have wrapped these details in comments so you can easily identify them:

Interactive Booking Wizard

The Live Appointment Booking Wizard is driven by Template/assets/js/booking.js. The wizard operates in three key steps:

  1. Select Care: User picks a service card and a practitioner card. The scripts detect validation conditions before allowing navigation to step 2.
  2. Choose Date & Time: Renders a dynamic calendar showing the next 14 open days, filtering out days the chosen doctor does not work. Clicking a day queries available hourly slots.
  3. Patient Info: Standard contact details form. Submitting will trigger a client-side mock loader overlay and output a complete transaction summary card displaying unique confirmation reference codes.
Customizing form handler: To connect the final booking output with an email engine (e.g. PHP mailer) or an API, customize the submitAppointment() function inside Template/assets/js/booking.js.

Accessibility (WCAG AA Compliance)

Zenith Dental strictly implements WCAG AA criteria to provide complete support for screen readers and keyboard navigation:

  • Focus Trapping: When the details modal is active, focus navigation is trapped inside the popup container, and focus is restored to the initiating button on closing.
  • Dynamic ARIA: Calendar day cells compute and set detailed descriptive titles (e.g. aria-label="Wednesday, July 15, 2026") so assistive screen readers read complete context instead of simple integers.
  • Contrast Ratios: Colors like --color-text-light are set to Slate 500 (#64748b) to guarantee contrast scores over 4.5:1 on white backdrops.

SEO & Performance Optimizations

  • JSON-LD Structured Data: Configured detailed Dentist structured metadata in the head of index.html to allow search engine crawlers to parse address, hours, geo-coordinates, and social links instantly.
  • WebP Format: All clinician profiles and clinic images have been converted to high-performance WebP formats, decreasing asset payloads by roughly 80% to ensure Lighthouse load speeds over 95%.
  • Lazy Loading: All card graphics below the fold are loaded asynchronously using loading="lazy" tags.
